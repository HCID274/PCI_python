# PCI项目图片命名规范

## 统一命名格式

所有主要图片文件应使用以下统一格式：

```
fig{number}_{description}_t{time:.2f}.png
```

## 标准图片类型

| 编号 | 图片类型 | 文件名格式 | 说明 |
|------|----------|------------|------|
| 1 | 3D光束几何图 | `fig1_beam_geometry_t{time:.2f}.png` | 显示光束的3D几何形状和托卡马克边界 |
| 2 | 密度poloidal截面图 | `fig2_density_poloidal_t{time:.2f}.png` | 密度场的poloidal截面可视化 |
| 3 | 检测器信号图 | `fig3_detector_signal_t{time:.2f}.png` | 检测器信号的等高线图 |
| 4 | 波数空间图 | `fig4_wavenumber_space_t{time:.2f}.png` | 2D波数空间分析结果 |

## 时间格式

- 时间使用两位小数格式：`{time:.2f}`
- 例如：`t98.07`, `t0.00`, `t45.32`

## 文件位置

- 所有图片保存在：`PCI/Data/sample/output/figures/` 目录下
- 或通过 `path_config.figures_dir` 指定的目录

## 特殊说明

1. **分析图片**：FFT分析等过程中生成的中间图片可以使用不同的命名规则，如 `mode1_kxky_f{f_idx}.png`
2. **批量处理**：对于时间序列分析，可以生成多个时间点的图片
3. **测试图片**：测试和示例代码可以生成特定命名的图片

## 命名示例

```
fig1_beam_geometry_t98.07.png
fig2_density_poloidal_t98.07.png  
fig3_detector_signal_t98.07.png
fig4_wavenumber_space_t98.07.png
```

## 错误示例（避免使用）

❌ `figure1_3d_beam_geometry_t98.07.png` (使用"figure"而不是"fig")
❌ `beam_geometry.png` (缺少编号和时间信息)
❌ `detector_signal_t{t_idx}.png` (时间格式不统一)
